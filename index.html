Examples of button variations using classes. Change any button color by passing a $color into the mixin. Set to a border style by passing $outline:true into the mixin, or adding .button-outline to the markup. Buttons and links have different default padding and text alignment, so if you are using a mixin on the fly rather than a class on the markup (as seen below) you may need to pass $type:link into mixin. Forms

This is a standard form layout with standard field and label styling. Every form will probably need a little bit of custom tweaking in order to get the padding right, as you can see if you inspect this form and it's elements.

Every form component has a wrapper ".fc-item" to which the element and component is aligned. Inspect the markup to see how these are each built.

Limitations of Use: This is the default form styling. When is doubt, use this layout ;)

Where this sample falls short: I wasn't sure how to accomplish this, so I'd love some FE Engineer help. You will see when you click on an input field and enter text, once the field is filled in, the text goes from italic to roman. This is easily accomplished with inputs by styling the placeholder text separately from the field text. I'm not sure how to do this with Select boxes, since they only have options. Is there some way to detect when a select option other than the default (empty) option is chosen, so that the same behavior can apply?

Default State Colors: Labels are dark-gray, placeholders are medium-gray, entered values are dark-gray.

Active / Focus State Colors: placholders and values are blue. Labels stay dark-gray. See section on disabled form component styles for those colors.


This is a form with inline labels. Add the class ".has-inline-label" to the .fc-item wrapper.

Limitations of Use: This layout should ONLY be used when the form is laid out in column, and is only a single form component per row.

This is a form with alternate styles. Add the class ".fc-alt" to the .fc-item wrapper.

Limitations of Use: This layout should ONLY be used for contact forms, or very short forms (less than 6 fields), and the form may only be *AT MOST 2 columns wide / 2 form components per row.

Disabled form components: Disabled form components have a soft-white background-color, labels and placeholder text are gray. Labels are italicized.
